# 🚀 Quick Start - AI Interview Platform

## One-Command Setup

```bash
./setup.sh && ./configure-api-keys.sh && ./start.sh
```

That's it! The application will open at http://localhost:3002

---

## Manual Commands

### Setup (First Time Only)
```bash
./setup.sh
```
> This installs dependencies, creates the database, generates a secure `JWT_SECRET`,
> sets `CORS_ALLOWED_ORIGINS`, and builds the backend. You only need to run it once.

### Configure API Keys
```bash
./configure-api-keys.sh
```

### Start Services
```bash
./start.sh
```
> The start script automatically loads `backend/.env`, validates that all required
> environment variables are set, and starts Ollama → Backend → Frontend in order.

### Stop Services
```bash
./stop.sh
```
> Uses graceful SIGTERM shutdown (waits up to 10 s) before force-killing.

## What You Need

### Required Services
- ✅ MySQL (database)
- ✅ Ollama (AI question generation)

### Required API Keys
- ✅ ElevenLabs (Primary TTS) — https://elevenlabs.io/
- ✅ D-ID (Fallback Avatar) — https://www.d-id.com/
- ✅ AssemblyAI — https://www.assemblyai.com/

### Required Environment Variables (No Defaults — Backend Fails Fast)

| Variable | Purpose | Set By |
|----------|---------|--------|
| `JWT_SECRET` | Signs auth tokens (≥ 64 chars, HS512) | Auto-generated by `setup.sh` |
| `CORS_ALLOWED_ORIGINS` | Frontend origin for CORS | Auto-set to `http://localhost:3002` by `setup.sh` |
| `DB_USER` | MySQL username | Set in `backend/.env` |
| `DB_PASSWORD` | MySQL password | Set during `setup.sh` or `configure-api-keys.sh` |
| `ELEVENLABS_API_KEY` | Text-to-Speech API | Set via `configure-api-keys.sh` |
| `DID_API_KEY` | Avatar video generation API | Set via `configure-api-keys.sh` |
| `ASSEMBLYAI_API_KEY` | Speech-to-Text API | Set via `configure-api-keys.sh` |

> ⚠️ After the security audit, `JWT_SECRET` and `CORS_ALLOWED_ORIGINS` have **no
> default values**. If either is missing or blank, the backend will refuse to start
> with an `IllegalStateException`. Run `./setup.sh` to auto-generate them.

## Troubleshooting

### Backend crashes immediately on startup

**Symptom:** `IllegalStateException: JWT_SECRET must be set` or
`CORS_ALLOWED_ORIGINS must be set and non-blank`

```bash
# Check that required env vars are present and non-empty
grep -E '^(JWT_SECRET|CORS_ALLOWED_ORIGINS|DB_USER|DB_PASSWORD)=' backend/.env

# If JWT_SECRET is missing or empty, regenerate it:
NEW_SECRET=$(openssl rand -base64 64 | tr -d '\n')
sed -i.bak "s|^JWT_SECRET=.*|JWT_SECRET=${NEW_SECRET}|" backend/.env && rm -f backend/.env.bak

# If CORS_ALLOWED_ORIGINS is missing:
echo "CORS_ALLOWED_ORIGINS=http://localhost:3002" >> backend/.env
```

### Backend won't start (other reasons)

```bash
# Check if MySQL is running natively or via Docker Desktop
docker-compose ps
mysql -u root -p -e "SELECT 1"

# Check if port 8081 is already in use
lsof -ti:8081

# Check if Ollama is running
curl http://localhost:11434/api/tags

# Check all environment variables
cat backend/.env
```

### Frontend won't start

```bash
# Reinstall dependencies
cd frontend
rm -rf node_modules package-lock.json
npm install --legacy-peer-deps
```

### CORS errors in the browser

```bash
# Verify CORS configuration matches the frontend URL
grep CORS_ALLOWED_ORIGINS backend/.env
# Must be: CORS_ALLOWED_ORIGINS=http://localhost:3002
# (no trailing slash, no wildcard *)
```

### "Unauthorized" or logged out unexpectedly

After the security audit, auth tokens are stored in **sessionStorage** (not
localStorage). This means:

- Tokens are **cleared when you close the browser tab** — this is intentional
  for security (reduces XSS token exfiltration window).
- Each browser tab has its own session — opening a new tab requires re-login.
- There is **no refresh token** yet — sessions expire after 1 hour.

## Quick Commands Reference

```bash
# ── Setup & Start ─────────────────────────────────────────────
./setup.sh                 # First-time setup (generates JWT_SECRET, etc.)
./configure-api-keys.sh    # Interactive API key configuration
./start.sh                 # Start all services (sources backend/.env automatically)
./stop.sh                  # Stop all services (graceful shutdown)

# ── Health Checks ─────────────────────────────────────────────
curl http://localhost:8081/actuator/health   # Backend health
curl http://localhost:11434/api/tags         # Ollama status
curl -s http://localhost:3002 | head -5      # Frontend status

# ── Logs ──────────────────────────────────────────────────────
tail -f logs/backend.log
tail -f logs/frontend.log

# ── Restart Individual Services ───────────────────────────────
# Restart backend only (stop old, source env, start new):
kill -15 $(lsof -ti:8081) 2>/dev/null; sleep 3
set -a; source backend/.env; set +a
cd backend && nohup ./mvnw spring-boot:run > ../logs/backend.log 2>&1 &
cd ..

# Restart frontend only:
kill -15 $(lsof -ti:3002) 2>/dev/null; sleep 2
cd frontend && nohup npm start > ../logs/frontend.log 2>&1 &
cd ..

# ── Validate Environment ─────────────────────────────────────
grep -cE '^(JWT_SECRET|CORS_ALLOWED_ORIGINS|DB_USER|DB_PASSWORD)=.+' backend/.env
# Should output: 4  (all four vars present and non-empty)
```

## Environment Variables

Edit `backend/.env` to configure:

```bash
# ── Database ──────────────────────────────────────────────────
DB_HOST=localhost
DB_PORT=3306
DB_NAME=interview_platform
DB_USER=root
DB_PASSWORD=your_password

# ── JWT (REQUIRED — no default, app fails fast if missing) ────
# Auto-generated by setup.sh. Must be ≥ 64 characters for HS512.
# Regenerate with: openssl rand -base64 64
JWT_SECRET=<auto-generated-by-setup>

# ── CORS (REQUIRED — no default, app fails fast if missing) ───
# Must match the exact frontend origin (no trailing slash, no wildcard)
CORS_ALLOWED_ORIGINS=http://localhost:3002

# ── Storage ───────────────────────────────────────────────────
# Use an absolute path in production (e.g. /var/data/uploads)
STORAGE_PATH=./uploads

# ── API Keys ──────────────────────────────────────────────────
ELEVENLABS_API_KEY=your_key
DID_API_KEY=your_key
ASSEMBLYAI_API_KEY=your_key
```

## Security Notes (Post-Audit)

- **JWT storage:** Tokens are now stored in `sessionStorage` (not `localStorage`)
  to reduce XSS exposure. Tokens are cleared when the tab closes.
- **No wildcard CORS:** `CORS_ALLOWED_ORIGINS=*` is rejected at startup.
  You must specify explicit origins.
- **File uploads:** Max 10 MB (was 20 MB). PDF and DOCX only, validated by
  magic bytes (not just file extension).
- **Rate limiting:** 5 requests per 60 seconds on expensive endpoints
  (start interview, complete interview, upload resume).
- **Graceful shutdown:** `./stop.sh` now sends SIGTERM first and waits 10 s
  before SIGKILL, allowing Spring Boot to close DB connections cleanly.
